## 目前判断数据类型

​	下面讨论判断数据类型,主要指的是字段名,字段注释等无法提供有效信息,只能通过数据值的一些特征来帮助分析数据可能的情况,如这个字段可不可能是个代码,或者看他的值像不像是个日期时间字段,或者是身份证号,手机号一些通用的字段。

### 判断代码

​	之前判断代码都是将该字段分组统计出来每个字段值以及该字段值出现的次数，这样的方式来看这个字段是不是代码，如果空值出现的比较多，或者近乎是的时候，判断代码的也会带来不小的难度。

### 判断日期时间

​	如果字段类型是date等日期类型，都不需要做这些操作，如果字段类型为string类型,就要取出数据来具体看数据值到底是什么，偶尔出现类型美式时间都会差异一下。

### 判断关键字段

​	在注释不明的情况下，某些关键字段所具备的特征就是水中月一样，非常显眼，如身份证号,手机号等既关键又重要上面列举的情况部分可能字段注释会提供一些帮助，或者字段类型提供一些帮助，但是现场大多数的情况时，他们提供的帮助有限。



​	通过上面来看要不需要统计每个值对应出现的次数,看看它是不是像代码,要不然就要看看这个值，是不是常用的字段的情况。

​	那么它们到底有没有共性帮助判断信息。

​	

## 数据组成定义

| 代码值 | 含义                                |
| ------ | ----------------------------------- |
| 0      | 空值，或者未必条件覆盖的值          |
| 1      | 代表特殊符号,主要指的是小写英文符号 |
| 2      | 代表大写小写字母                    |
| 4      | 代表数字                            |
| 8      | 代表汉字或者是多字节词组            |

​	它们也只是定义了基础，如果出现了字母和特殊符号组成的类型，那么就定义为3,这个类型可以叠加，帮助分析数据值的组成部分,(其实这里有个隐含假设,就是数据的组成和数据的顺序一致,在现实场景下这个假设是大多数情况下是错误的,在这里先要认为是准确的)。

​	那么共计类型会有16种类型刚好是2的四次方,在后面说的数据元中,会发现有很多数据元的数据组成是单一类型,这里可以指出来如代码类型,号码类型。而对于描述字段中如备注，他的组成成分就会非常多，具体以谁的类型为高判断要到后面去讨论。

​	这是一段时间做项目过程发现的现象。	

​	之前在做数据对标时，经常通过字段注释对完后，还需要去数据库看看数据的真实情况，如果仔细的去核实，会发现一部分字段注释或者匹配出来的数据元问题，这样可能提高了准确率但是效率会明显下降。

​	在做一些数据质量管理的项目时发现，很多字段值不符合字段含义要求就是数据组成不对(其他情况也有存在)。不过相等于数据值不符合数据含义，那么数据组成不符合数据含义反而更加突出。

​	~~那么为什么数据格式不符合规范远远高于数据值不符合规范(这个暂时是部分现象,没有明显的证明,感觉是人的认知)~~



### 数据组成可以帮助判断什么

- 如果已经对标出现对应的数据元，那么可以和数据元结合，初步帮助判断数据元与数据值是否保持一致，如果不一致，那么需要检查数据元，或者参考字段注释字段名来综合考量一下该字段数据质量问题。

- 如果暂时没有对出数据元，那么可以帮助一定程度上判断数据元的大致范围,减少时间上的一些浪费。

  

  当然仅仅是有数据组成其实时不够多的，判断代码等信息还需要字段长度的要求，因为本身字段的组成就是字段类型加上字段长度的。



## 分析数据组成

​	分析数据组成

​	分析数据组成经常会发现在人名中，出现汉字的会远远大于出现汉字加上特殊字符的，但是汉字加上特殊字段对于这个数据元的定义来说并不代表不对的。但是对于这个值来说，当出现什么情况下，这些数据值可以舍弃，当数据值在多少时,数据值反馈出来的情况能够被采纳

​	是一种类型占据非常大，在100%，还是只要在非常小，只需要在10%，这个比例是在多少既能够尽量接收最大的字段分析又能可以比较好的反馈字段真实情况？

​	这是描述每种类型在每个字段的占比统计数据,如某个类型在字段A的组成超过了90就在95上面加个1

​	通过图片可以明显的看出，数据呈现两极分化趋势，那么考虑数据组成在90%~100%的话，对于80%~100%的增加程度一点都不明显，那么使用90%~100%对于分析数据明显优于80%~100%的准确性

​	对于数据来说,那么选择90%~100%就非常高的可能性,接近1/4的可能性。其实这从侧面很多字段只有一种字段类型，如那种性别代码,通常会出现字段长度为1,字段类型是数字型,出现其他的情况可能就是异常数据，不过不在讨论范围，那么那种如备注字段的话，可能在一个字段中汉字也有,数字也有,字母也有,符号可能也有,那么各种类型组合在一起就非常多的可能性了。

### 猜测两个90%保持相交并且概率高

```
select count(1) from YSYS_TMP_1118_02 t,YSYS_TMP_1118_01 h  where t.bsf_getf0 > 90 and h.bsf_getf0 >90 and
t.bm = h.bm and t.zdm = h.zdm and t.xzqh = h.xzqh;

--13416

select count(1) from YSYS_TMP_1118_01  t where t.bsf_getf0 > 90
--19389
select count(1) from YSYS_TMP_1118_02  t where t.bsf_getf0 > 90
--13825
```

​	发现数据情况是字段组成和字段长度都在90%共有13416个,接近13825个,当然与字段组成的还是接近1/3差距了了。这个情况大致上对于我们研判数据类型就有非常高的可靠性。

​	就是数据组成如果占据全部数据的90%，而且在这个数据组成中字段长度能够占据在90%左右，通过限制一定的长度能够更好的标注出来代码或者其他定长的字段。



### 实践90%猜测数据类型



​	考虑到数据组成是人为对于各种组成进行了数字排序，不太适合使用一些数据统计脚本，那么只设定如下定义超过90%,或者定义10%的组成,其他可能各种情况都会发生

​	考虑字段的长度情况，那么对于是否可以一些统计信息，如方差,标准差,平均值这种是否适合使用？

​	也就是说先统计90%的字段组成情况,然后分析字段长度情况,字段组成下的字段长度分析。

​	因此首先统计出来字段组成和字段长度都超过90%的单一类型定长的情况，很大程度上和数据元的定义是相关的(数据元大部分是抽象或者直接引用数据)。其他情况暂时将详细情况写到了备注上。

### 代码判断规则

​	说实话对于代码判断规则暂时是无法说清的,毕竟是每个人的思路不同,代码编写规则就有差异,不过暂时先以如下标准来确定

```
1、字段组成超过样本数据的90%
2、在样本超过90%的字段组成中找到超过90%
3、样本唯一值计算规则 1/样本唯一值个数 > 1/1000(大部分代码很难超过1000次)
4、样本唯一值计算规则 样本唯一值个数/样本总数<1/200(默认5%非常难出现,考虑到数据离散情况,默认除以10)
5、上面中字段长度不能超过6，如果超过6的话修改计算规则
6、样本唯一值计算规则 1/样本唯一值个数 > 1/2000
7、样本唯一值计算规则 样本唯一值个数/样本总数<1/100(默认1%非常难出现)
```

## 数据元

​	数据元按照BSC分层的话，主要划分为代码，日期，号码，描述，量，日期时间，名称，金额等。

​	那么分别对应着有哪些可能的情况

|          | 特殊符号 | 字母   | 数字   | 汉字   | 备注                                                         |
| -------- | -------- | ------ | ------ | ------ | ------------------------------------------------------------ |
| 代码     |          | 可能有 | 可能有 |        | 可能有字母，可能是数字，可能是两者组合                       |
| 日期     |          |        | 有     |        | 本次讨论的是YYYYMMDD类型                                     |
| 号码     |          | 可能有 | 可能有 |        | 可能有字母，可能是数字，可能是两者组合，不讨论带有特殊符合的可能性 |
| 描述     | 可能有   | 可能有 | 可能有 | 可能有 | 描述字段的字段类型分布比较乱，下面会专门讨论这一个题目       |
| 量       | 可能有   |        | 有     |        | 主要指的是量，可能有特殊符号                                 |
| 日期时间 |          |        | 有     |        | 本次讨论的是YYYYMMDDHH24MISS类型                             |
| 名称     |          |        |        | 可能有 | 对于名称仅是讨论可能是汉字组成                               |
| 金额     | 可能有   |        | 可能有 |        | 可能有特殊符号，可能有数字                                   |

​		



## 猜测

​	对于数据字段的值来说,反应字段真实含义才是它的本质,那么它的数据分布就可以在一定的程度上划分出来了，如常说的姓名字段,移动电话字段,数据会在一定程度上反应这些信息,当然如果是某些其他原因导致如姓名字段,移动电话字段等不能反应数据字段的真实意图,现在只能归结到数据质量问题。
