# Greenplum:数据库SSH非默认端口22安装

**document support**

ysys

**date**

2018-12-30

**label**

linux,centos7.3,greenplum,4.x,ssh,22,other port



## 总结

​	之前查询文档发现gp6.x可以通过修改python脚本来规避ssh22端口造成的问题,但是在gp4.x系列中并没有发现，思考gpssh和gpscp以及gpssh-exkeys关系后，发现部分是可以跳过去的，例如，gpssh-exkeys生成公钥后直接拷贝到各个节点服务器上gpssh,gpscp就可以在节点上使用。

​	还有一点就是gpssh,gpscp,gpssh-exkeys与greenplum的运行没有直接关系，或者可以将gpssh-exkyes，gpssh,gpscp就是简便工具帮助来进行文件拷贝文件修改文件上传文件打包文件下载文件解压的功能。也就是说，如果实在是各种因素下，可以每台服务器安装一下文件再进行部署。



##  注意点

- 之前gp的安装路径默认都是/usr/local环境下,这次部署建议放到新的目录，并且该目录下可以给gpadmin用户授权
- 之前root用户和gpadmin用户同时做了免密码协议，本次只做gpadmin用户
- 本次文档使用gp4.x安装,gp4.x版本大致相同
- `#`代表root用户,`$`代表gpadmin用户



## 环境

​	operation version:centos 7.3 x64

​	software version:greenplum-db-4.2.6.0/greenplum-db-4.3.0



## 操作步骤

### 所有服务器执行命令

修改主机名

```
# vim /etc/hostname
```

修改sshd端口,默认需要重启服务sshd.service

```
# vim /etc/ssh/sshd_config
Port 10000
```

修改ssh端口,直接生效,如果不修改ssh端口,当sshd默认端口被修改后,命令会变为 ssh -p xxxxx root@xxx.xxx.xxx.xxx

```
# vim /etc/ssh/ssh_config
Port 10000
```

关闭防火墙和安全策略

```
# systemctl stop firewalld.service 
# systemctl disable firewalld.service 
# vim /etc/selinux/config
disabled
```

修改主机映射

```
# vim /etc/hosts

192.168.1.105 master
192.168.1.106 slave1
192.168.1.107 slave2
```

修改内核参数

```
# vim /etc/sysctl.conf

kernel.shmmax = 500000000
kernel.shmmni = 4096
kernel.shmall = 4000000000
kernel.sem = 250 512000 100 2048
kernel.sysrq = 1
kernel.core_uses_pid = 1
kernel.msgmnb = 65536
kernel.msgmax = 65536
kernel.msgmni = 2048
net.ipv4.tcp_syncookies = 1
net.ipv4.ip_forward = 0
net.ipv4.conf.default.accept_source_route = 0
net.ipv4.tcp_tw_recycle = 1
net.ipv4.tcp_max_syn_backlog = 4096
net.ipv4.conf.all.arp_filter = 1
net.ipv4.ip_local_port_range = 1025 65535
net.core.netdev_max_backlog = 10000
vm.overcommit_memory = 2

```

**kernel_shmmax值建议是物理内存的一半 1*1024*1024*1024**

参数生效

```
# sysctl -p /etc/sysctl.conf
```

修改限制文件信息

```
#  vim /etc/security/limits.conf
* soft nofile 65536
* hard nofile 65536
* soft nproc 131072
* hard nproc 131072
```

添加用户，创建文件并授权

```
# groupadd gpadmin
# useradd -g gpadmin gpadmin -p gpadmin
# mkdir -p /data/greenplum/
# chown -R gpadmin:gpadmin /data
```

### 主机安装命令

安装命令

```
# cd /software/
# unzip greenplum-db-4.3.7.2-build-2-RHEL5-x86_64.zip 
# chmod +x greenplum-db-4.3.7.2-build-2-RHEL5-x86_64.bin 
# ./greenplum-db-4.3.7.2-build-2-RHEL5-x86_64.bin 
```

默认路径安装到了/usr/local下，选择路径是填写/data/greenplum/greenplum-db-4.3.7.2

```
# cd /data/greenplum/
# chown -R gpadmin:gpadmin *

-- gpadmin用户下
# su - gpadmin
$ cat >> /home/gpadmin/.bash_profile << EOF
> source /data/greenplum/greenplum-db/greenplum_path.sh
> EOF
$ source .bash_profile
```

利用gpscp,gpssh可以在数据节点上执行命令将文件传递过去，创建文件

```
$ mkdir -p /data/master
$ cd /data/greenplum/
$ tar cf gp.4.3.tar greenplum-db-4.3.7.2
$ gzip gp.4.3.tar
$ gpscp -f /home/gpadmin/all_segs gp.4.3.tar.gz gpadmin@=:/data/greenplum/gp.4.3.tar.gz
$ gpscp -f /home/gpadmin/all_segs /home/gpadmin/.bash_profile gpadmin@=:/home/gpadmin/.bash_profile
```

更新环境信息

```
$ gpssh -f /home/gpadmin/all_segs -e "source /home/gpadmin/.bash_profile"
```

修改配置文件

```
$ vim /home/gpadmin/gpinitsystem_config 
ARRAY_NAME="EMC Greenplum DW"
SEG_PREFIX=gpseg
PORT_BASE=40000
declare -a DATA_DIRECTORY=(/data/primary1 /data/primary2)
MASTER_HOSTNAME=master
MASTER_DIRECTORY=/data/master
MASTER_PORT=5432
TRUSTED_SHELL=ssh
CHECK_POINT_SEGMENTS=8
ENCODING=UNICODE
MIRROR_PORT_BASE=50000
REPLICATION_PORT_BASE=41000
MIRROR_REPLICATION_PORT_BASE=51000
declare -a MIRROR_DATA_DIRECTORY=(/data/mirror1 /data/mirror2)
DATABASE_NAME=gpdw
MACHINE_LIST_FILE=/home/gpadmin/all_segs
```

初始化数据库

```
$ gpinitsystem -c /home/gpadmin/gpinitsystem_config 
```

配置参数查看数据库环境

```
$  export MASTER_DATA_DIRECTORY=/data/master/gpseg-1

$ gpstate -m
```
