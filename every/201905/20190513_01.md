## 前期说明

- 目前数据治理主要涉及的是数据对标，因此上文可以改为数据对标整体流程
- 目前将部分省市的对标进行了汇总，形成了历史对标库
- 针对历史数据库原则时，尽量使用更新日期靠前的对应规则
- 在测试中发现对标完成率在70%-80%中间，错误在10%左右
- 后期。。。



## 遗留问题

- 本次整理的对标需要依赖postgresql分词，为了更好的使用，已将oracle与postgresql同步整理到kettle工具中，postgresql可以使用库备份直接使用
- postgresql分词需要依赖zhparser插件，整体安装较为简单，参考链接[postgresql chinese participle](20190505_01.md)
- 本次涉及到将数据元从大变小的过程，有很多数据元无法使用，有一部分数据元无法对应新的数据元，可能后期匹配会出现无法匹配中文名称
- 同义词或者叫做接近同义词，尽量往长字段去取，这是为了更好准确匹配数据元(如工厂工作人员，同义词建议使用工作人员映射姓名,而非人员映射姓名)
- 本次治理中添加了判断数据元与注释是否可能匹配，只能大概解决数据映射快速检查
- 本次治理中没有添加对注释的分析，如注释中如下信息`性别 1汉族`这样单独处理，而是模糊匹配
- 待续

####  插入数据

```



merge into sjzl_history_data t
using (select T.注释 as annotate,
              t.数据元编号 as standar_source,
              t.限定词 as limit_cn_name,
              t.限定词缩写 as limit_en_name,
              sum(t.sjl) as fenquency
         from TMP_GH_0612_01 t
        GROUP BY T.注释, t.数据元编号, t.限定词, t.限定词缩写) h
on (t.annotate = h.annotate and t.standar_source = h.standar_source and nvl(t.limit_cn_name, 'gh') = nvl(h.limit_cn_name, 'gh'))
when matched then
  update
     set t.fenquency   = t.fenquency + h.fenquency,
         t.program     = t.program + ?(2的n次方),
         t.update_time = trunc(sysdate)
when not matched then
  insert
    (t.id,
     t.annotate,
     t.standar_source,
     t.limit_cn_name,
     t.limit_en_name,
     t.fenquency,
     t.clear_flag,
     t.program,
     t.insert_time,
     t.update_time)
  values
    (seq_id.nextval,
     h.annotate,
     h.standar_source,
     h.limit_cn_name,
     h.limit_en_name,
     h.fenquency,
     '',
     64,
     trunc(sysdate),
     trunc(sysdate));




UPDATE TMP_GH_0612_01 H 
SET H.限定词缩写=UPPER(F_GETPY(H.限定词))
WHERE (h.注释,h.数据元编号,h.限定词) in (
select  annotate,standar_source,limit_cn_name from (select T.注释 as annotate,
              t.数据元编号 as standar_source,
              t.限定词 as limit_cn_name,
              t.限定词缩写 as limit_en_name,
              sum(t.sjl) as fenquency
         from TMP_GH_0612_01 t
        GROUP BY T.注释, t.数据元编号, t.限定词, t.限定词缩写) group by annotate,standar_source,limit_cn_name having count(1)>1);
```
