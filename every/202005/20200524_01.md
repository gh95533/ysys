## 4.1 数值类型

​	数值类型包括整数类型，任意精度数字类型，浮点类型，序列类型，货币类型等。



### 4.1.1 整数类型

​	smallint,integer,bigint都是整数类型，每种类型只能存储一定范围的整数，超过该范围就会报错。

| 名字     | 存储容量 | 说明           | 范围                                     |
| -------- | -------- | -------------- | ---------------------------------------- |
| smallint | 2字节    | 小范围整数     | -32768~32767                             |
| integer  | 4字节    | 整数的常见选择 | -2147483648~2147483647                   |
| bigint   | 8字节    | 大范围整数     | -9223372036854775808~9223372036854775807 |

​	常见的整数类型integer，因为它提供了在范围，存储空间和性能之间的最佳平衡

​	一般只有在磁盘容量不足的时候才使用smallint。

​	而只有integer类型的范围不够时才能bigint类型

【举例】

```
ysys=# CREATE TABLE test_integer(id1 integer,id2 smallint,id3 bigint);
CREATE TABLE
ysys=# INSERT INTO test_integer VALUES(32400,1024,2147483648);
INSERT 0 1
ysys=# select * from test_integer ;
  id1  | id2  |    id3     
-------+------+------------
 32400 | 1024 | 2147483648
(1 row)
```



### 4.1.2 任意精度数字类型

​	任意精度数字类型包括numeric和decimal。两者在使用上是等价的



| 名字    | 存储容量 | 说明               | 范围                                      |
| ------- | -------- | ------------------ | ----------------------------------------- |
| numeric | 可变     | 用户指定精度，精确 | 最高小数点前131072位，以及小数点后16383位 |
| decimal | 可变     | 用户指定精度，精确 | 最高小数点前131072位，以及小数点后16383位 |

​	numeric类型通过numeric(precision,scale)来定义

- precision称为精度，表示总共的位数

- scale称为比例，表示小数点部分的位数

  如果用户插入数据小数位数超过scale，则系统自动会进行四舍五入，如果用户插入的数据超出最大范围，则插入失败

【举例】

```
ysys=# CREATE TABLE test_num(id numeric(6,2),id2 decimal(4,0));
CREATE TABLE
ysys=# INSERT INTO test_num VALUES(1024.44,2.3);
INSERT 0 1
ysys=# SELECT * FROM test_num ;
   id    | id2 
---------+-----
 1024.44 |   2
(1 row)
```



### 4.1.3 浮点类型

​	PostgreSQL中的浮点类型用来存储小数，浮点类型包括real和double precision。

| 名字             | 存储容量 | 说明             | 范围           |
| ---------------- | -------- | ---------------- | -------------- |
| real             | 4 字节   | 可变精度，不精确 | 6位十进制精度  |
| double precision | 8 字节   | 可变精度，不精确 | 15位十进制精度 |

​	在大部分平台中，real类型范围是-1E+37~1E+37,精度至少是6位数字，double precision类型的范围是-1E+308~1E+308,精度至少是15位精度，太大或者太小都会导致报错

​	PostgreSQL还支持SQL标准表示法float和float(p),用于声明非精确的数字类型。在这里，p指定以二进制位表示的最低可接受精度。

​	real和double precision都是不准确的，可变精度的数字类习惯，这意味着一些值不能精确地转换成内部格式，而是以近似的形式存储的。因此，存储和检索一个值可能出现一些缺失。如果要进行准确的数据存储和计算，则应该使用numeric类型。

【举例】

```
ysys=# CREATE TABLE test_fp(id1 real,id2 double precision,id3 float(8));
CREATE TABLE
ysys=# INSERT INTO test_fp VALUES(1024,2.3,4.72345789);
INSERT 0 1
ysys=# INSERT INTO test_fp VALUES(1024.4455,2.3,4.7);
INSERT 0 1
ysys=# SELECT * FROM test_fp ;
   id1   | id2 |   id3   
---------+-----+---------
    1024 | 2.3 | 4.72346
 1024.45 | 2.3 |     4.7
```

​	在PostgreSQL中，浮点类型有3个特殊值，即Infinity,-Infinity,NaN，分别代表正无穷大，负无穷大，非数字





### 4.1.4 序列类型

​	smallserial,serial和bigserial不是真正的数据类型，它们作为唯一标识符列而存在，用户在需要使用自增整数时，可以选用这三者之一。如果一个表中本身的字段中，没有诗意作为主键的，则可以增加一个专门的字段并指定serial类型作为主键。

| 名字        | 存储容量 | 说明             | 范围                  |
| ----------- | -------- | ---------------- | --------------------- |
| smallserial | 2 字节   | 自动增加的小整数 | 1~32767               |
| serial      | 4 字节   | 自动增加的整数   | 1~2147483647          |
| bigserial   | 8 字节   | 自动增加的大整数 | 1~9223372036854775807 |

【举例】

```
ysys=# CREATE TABLE test_seq(id1 SERIAL,num INTEGER,id3 FLOAT(8));
CREATE TABLE
ysys=# INSERT INTO test_seq(num,id3) VALUES(3,4.7);
INSERT 0 1
ysys=# INSERT INTO test_seq(num,id3) VALUES(4,4.7);
INSERT 0 1
ysys=# SELECT * FROM test_seq;
 id1 | num | id3 
-----+-----+-----
   1 |   3 | 4.7
   2 |   4 | 4.7
(2 rows)
```



### 4.1.5 货币类型

​	money类型用于存储固定小数精度的货币数字，其小数的精度由数据库系统的lc_monetary参数设置决定。lc_monetary参数用来指定货币符号对应的地区，与时区概念类似

​	money类型可以接受很多输入格式，包括整数和浮点数，以及常用的货币格式,如“$1,000.00".输出通常是货币格式。不同区域的货币符号有所不同。

| 名字  | 存储容量 | 说明   | 范围                                           |
| ----- | -------- | ------ | ---------------------------------------------- |
| money | 8字节    | 货币额 | -9223372036854775808.08~9223372036854775807.07 |

​	lc_monetary参数的默认设置是"en_US.utf8"，其货币格式美元，如果要输出为人民币格式，则需要将lc_moneraty参数设置为`zh_CN.utf8`

```
ysys=# SELECT '52093.89'::money;
   money    
------------
 $52,093.89
(1 row)

ysys=# show lc_monetary;
 lc_monetary 
-------------
 en_US.UTF-8
(1 row)

ysys=# set lc_monetary='zh_CN.utf8';
SET
ysys=# SELECT '52093.89'::money;
    money    
-------------
 ￥52,093.89
(1 row)
```



### 4.1.6 使用数学运算符

​	PostgreSQL为很多类型提供了数学运算符

| 运算符 | 说明                 | 示例    | 结果 |
| ------ | -------------------- | ------- | ---- |
| +      | 加                   | 2+3     | 5    |
| -      | 减                   | 2-3     | -1   |
| *      | 乘                   | 2*3     | 6    |
| /      | 除(整数除法截断结果) | 4/2     | 2    |
| %      | 模(取余)             | 5%2     | 1    |
| ^      | 指数                 | 2^3     | 8    |
| \|/    | 平方根               | \|/25   | 5    |
| \|\|/  | 立方根               | \|\|/27 | 3    |
| !      | 阶乘                 | 5!      | 120  |
| !!     | 阶乘(前缀运算符)     | !!5     | 120  |
| @      | 绝对值               | @-5     | 5    |
| &      | 按位与               | 91&15   | 11   |
| \|     | 按位或               | 32\|3   | 35   |
| #      | 按位异或             | 17#5    | 20   |
| ~      | 按位求反             | ~1      | -2   |
| <<     | 按位左移             | 1<<4    | 16   |
| >>     | 按位右移             | 8>>2    | 2    |

​	按位运算符只能用于整数类型，而其他的运算符可以用于全部的数据类型。按位运算符还可以用于位串类型bit和bit varying.

​	数值类型类型还支持比较运算符。比较运算符也可以用于除数值外的其他类型。如字符串类型。所有比较运算符都是双目运算符，它们返回boolean类型

| 运算符   | 说明     |
| -------- | -------- |
| <        | 小于     |
| >        | 大于     |
| <=       | 小于等于 |
| >=       | 大于等于 |
| =        | 等于     |
| <> 或 != | 不等于   |



## 4.2 字符串类型



### 4.2.1 字符串类型分类

​	PostgreSQL提供了三种基础的字符串类型:character(char),character varying(varchar),text

| 名字       | 说明                                 |
| ---------- | ------------------------------------ |
| char(n)    | 固定长度字符串，长度不足时以空格补充 |
| varchar(n) | 有长度限制的变长字符串               |
| text       | 无长度限制的变长字符串               |

​	character(n)占用的存储空间是固定的，其中正整数n表示最多存储n个字符

- 如果输入的字符串长度超过n，则PostgreSQL会抛出报错。
- 如果字符串长度比n短，则数据库会自动使用空格填满，不管存储还是查询显示时都是如此。

   character varyning(n)和text都使用与存储长度可变的文本，根据每行记录的长度分配相应的空间，两者主要区别在于长度n的限制。对于character varyning(n)来说，正整数n同样表示最多存储n个字符的字符串。

【举例】

```
ysys=# CREATE TABLE test_char(a character(4),b varchar(5));
CREATE TABLE
ysys=# INSERT INTO test_char VALUES('ok','test');
INSERT 0 1
ysys=# SELECT * FROM test_char;
  a   |  b   
------+------
 ok   | test
(1 row)
```



### 4.2.2 使用连接运算符

​	连接运算符可以作用所用字符串类型，对于字符串类型来说，自动填充的空格对连接没有任何影响，连接时会被自动去掉。

| 运算符                                      | 返回类型 | 说明                       | 示例            | 结果     |
| ------------------------------------------- | -------- | -------------------------- | --------------- | -------- |
| string\|\|string                            | text     | 串接                       | ‘Post'\|\|'gre' | Postgre  |
| string\|\|non-string 或non-string\|\|string | text     | 使用一个非字符串输入的串接 | 'Value:'\|\|42  | Value:42 |

【举例】

```
ysys=# SELECT a,b,b||a FROM test_char;
  a   |  b   | ?column? 
------+------+----------
 ok   | test | testok
(1 row)

ysys=# SELECT a,a||4 FROM test_char;
  a   | ?column? 
------+----------
 ok   | ok4
(1 row)
```



### 4.2.3 使用模式匹配运算符

​	PostgreSQL提供了3种独立实现模式匹配的方法:LIKE 运算符,SIMILAR TO 运算符,POSIX 运算符

**1.LIKE 运算符**

​	LIKE运算符语法格式如下

```
string LIKE pattern [ESCAPE escape-character]
string NOT LIKE patttern [ESCAPE escape-character]
```

【举例】

```
ysys=# SELECT 'ABC' LIKE 'ABC';
 ?column? 
----------
 t
(1 row)

ysys=# SELECT 'ABC' NOT LIKE 'ABCD';
 ?column? 
----------
 t
(1 row)
```

​	LIKE运算符总是覆盖整个字符串。因此，要匹配在字符串中任何位置的序列，该模式必须以百分号开头或者结尾，在patttern中的下划线表示匹配的任何单个字符，百分号表示匹配0个或者多个字符的序列.

​	如果pattern不包含百分号或下划线，则改模式只表示它本身的字符串，这是，LIKE运算符的作用相当于等号运算符

​	为了匹配`_`和`%`字符本身，在相应的模式匹配字符串中，这两个字符必须冠以转义字符。

​	默认的转义字符是反斜线`\`,如果想使用其他转义字符，则需要使用ESCAPE子句进行声明。如果想匹配转义字符本身，则连续写两个转义字符。

【举例】

```
               ^
ysys=# SELECT 'abc_de%' like '%\_%';
 ?column? 
----------
 t
(1 row)

ysys=# SELECT 'abcde%fg' like '%\%%';
 ?column? 
----------
 t
(1 row)

ysys=# SELECT 'abcde\fg' like '%\\%' ESCAPE '|';
 ?column? 
----------
 f
(1 row)
```



**2.SIMILAR TO 运算符**

​	SIMILAR TO 运算符的语法格式如下

```
string SIMILAR TO pattern [ESCAPE escape-character]
string NOT SIMILAR TO pattern [ESCAPE escape-character]
```

​	SIMILAR TO 运算符根据自己的模式是否匹配字符串，而返回TRUE或者FALSE。

​	SIMILAR TO 运算符和LIKE运算符非常相似，只不是它是使用了SQL标准定义的正则表达式模式。

​	SIMILAR TO 运算符只能在它的模式匹配整个字符串时才能成功，这一点和普通正则表达式的行为不同
